# Technical Decisions

Use this folder for structured decision records that affect architecture, tooling, or long-term maintenance.

## Naming
1. Create files as zero-padded sequence numbers.
2. Format: `NNNN-short-title.md`.
3. Example: `0001-open-gl-baseline.md`.

## When to Add a Decision
1. Choosing between competing libraries or architecture approaches.
2. Accepting tradeoffs that influence multiple systems.
3. Defining standards other contributors must follow.

## Minimum Contents
1. Context.
2. Options considered.
3. Decision.
4. Consequences.
5. Follow-up actions.

## Lifecycle
1. New decisions start as `Proposed`.
2. Move to `Accepted` after review.
3. Mark `Superseded` with reference when replaced.
