# Mordor Overview

## Project Intent
Mordor has three primary goals:
1. Build a fun single-player party RPG.
2. Build a reusable engine foundation for later simulation projects.
3. Improve practical design and implementation skills.

## Design Direction
1. Ability-driven RPG gameplay where character capability matters more than player reflex.
2. Isometric 3D presentation with party-focused control.
3. Dungeon gameplay built around interactable world systems: doors, keys, switches, chests, traps, and containers.
4. Visibility and awareness model including line-of-sight, hearing, and fog of war.
5. Procedural generation that combines rule-based layouts and reusable prefabs.

## Technical Direction
1. Initial implementation target: C/C++ with OpenGL.
2. Linux-first development with portability constraints in architecture.
3. Data-first authoring workflow and deferred editor investment until schema maturity.
